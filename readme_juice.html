ðŸ§  The Concept

Modern web applications trust users way too much. They assume that if they hide a button, you won't click it. They assume that if they ask for a username, you won't type in a malicious database command instead. Your job is to break those assumptions.

You will be looking for things like:

  • Broken Authentication: Can you log in as the administrator without their password?
  • Cross-Site Scripting (XSS): Can you trick the website into running your own custom code?
  • Information Exposure: Did the developers leave sensitive files sitting on the server?

Juice Shop tracks your progress automatically, but the developers thought it would be funny to hide the actual Scoreboard from you. You cannot start tracking your hacks until you find it.

Getting Started:

  1. Open the website and look around. Inspect the navigation menus.
  2. Press F12 to open your browser's Developer Tools.
  3. Look at the Sources or Elements tab. The code running the website is fully visible to you.
  4. Try to find the hidden URL path for the scoreboard. Hint: What happens if you try to guess the URL by typing /scoreboard at the top of the browser?
