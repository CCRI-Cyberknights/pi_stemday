Welcome to SQLi-Labs! This is an interactive laboratory dedicated entirely to one of the most famous and dangerous hacks in history: SQL Injection.

ðŸ§  The Concept

Almost every website stores its user data, passwords, and products inside a database. When you type a username into a login box, the website takes your text and secretly inserts it into a database query (written in a language called SQL).

But what if you type a piece of SQL code instead of a normal username? If the developers didn't secure the website properly, the database will actually run your code! This allows hackers to dump credit card numbers, bypass logins, and steal entire user tables.

ðŸŽ¯ Your First Mission: Break the Query

This lab acts like an X-Ray. When you try to hack it, the screen will dynamically print exactly what the backend database is thinking, helping you learn how to craft the perfect attack.

Getting Started:

  1. Click the Setup/reset Database for labs button to build the vulnerable environment.
  2. Scroll down and click on Lesson 1.
  3. Look at the URL in your browser. Add ?id=1 to the end of the URL and press Enter. You should see a user's login name appear.
  4. Now, try to break the database! Add a single quote (') after the number 1 in the URL (like this: ?id=1').
  5. Read the error message on the screen. It is telling you exactly how the database code broke! Your goal is to figure out how to manipulate that error to show hidden data.
